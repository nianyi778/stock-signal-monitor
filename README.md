# Stock Signal Monitor

美股技术信号自动扫描 + Telegram 推送服务。每日收盘后扫描技术指标，**仅当多指标共振时**推送 LLM 生成的分析摘要，降低噪音干扰。

## 功能

- **多指标共振检测**：MACD、RSI、均线交叉、布林带四项指标，2+ 同向信号才算强信号
- **分级推送**：强信号（STRONG）推 Telegram，弱信号（WEAK/WATCH）仅入库保留历史
- **LLM 分析摘要**：GPT-4o-mini 生成中文分析（综合判断 + 操作建议 + 关键理由）
- **每日自动扫描**：APScheduler 在美股收盘后（17:00 ET）自动触发
- **Swagger UI 管理界面**：无需前端，直接通过 `/docs` 管理自选股、查看信号

## 快速上手

### 1. 准备配置文件

```bash
cp .env.example .env
```

编辑 `.env`，填入以下配置：

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | TiDB Serverless 连接串（MySQL 兼容） |
| `TELEGRAM_BOT_TOKEN` | Telegram Bot Token（从 @BotFather 获取） |
| `TELEGRAM_CHAT_ID` | 接收推送的 Chat ID |
| `OPENAI_API_KEY` | API Key（通过 Sub2API 代理使用 GPT-4o-mini） |
| `OPENAI_BASE_URL` | Sub2API 代理地址，默认已填 |
| `SCHEDULER_CRON_HOUR` | 每日扫描时间（小时，美东时区），默认 17 |
| `PUSH_MIN_CONFIDENCE` | 推送置信度阈值，默认 60 |

### 2. 启动服务

```bash
docker compose up --build
```

Apple Silicon 用户（OrbStack）构建 amd64 镜像：

```bash
docker buildx build --platform linux/amd64 -t stock-signal-monitor:1.0.0 .
docker compose up
```

### 3. 添加自选股

打开 [http://localhost:8000/docs](http://localhost:8000/docs)，使用 Swagger UI：

```
POST /api/stocks/
{"ticker": "AAPL"}

POST /api/stocks/
{"ticker": "NVDA"}
```

### 4. 手动触发扫描（测试用）

```
POST /api/stocks/scan
```

### 5. 查看信号

```
GET /api/signals/?level=STRONG     # 仅强信号
GET /api/signals/?limit=20         # 最近20条
GET /api/signals/AAPL              # 指定股票
```

## 信号逻辑

```
单指标触发   →  WEAK  （入库，不推送）
2+ 指标共振  →  STRONG（入库 + LLM 摘要 + Telegram 推送）
布林带单独   →  WATCH （入库，不推送）
置信度 < 60  →  不推送（即使 STRONG）
```

### 共振示例

| 指标状态 | 结果 |
|---------|------|
| MACD 上穿零轴 | WEAK BUY |
| RSI 超卖（<30） | WEAK BUY |
| MACD + RSI 同时触发 | **STRONG BUY**（indicator: "MACD+RSI"，置信度加成 +20） |

## 项目结构

```
app/
├── main.py            # FastAPI 入口 + lifespan
├── config.py          # 配置（Pydantic BaseSettings）
├── database.py        # SQLAlchemy + TiDB Serverless
├── models.py          # ORM: WatchlistItem, Signal
├── schemas.py         # Pydantic 响应模型
├── scheduler.py       # APScheduler 每日任务
├── api/
│   ├── stocks.py      # CRUD 自选股 + 触发扫描
│   └── signals.py     # 查询信号历史
├── data/
│   └── fetcher.py     # yfinance 数据获取
├── signals/
│   ├── indicators.py  # 纯函数：MACD/RSI/MA交叉/布林带
│   └── engine.py      # 信号引擎：检测 + 共振合并
├── llm/
│   └── summarizer.py  # GPT-4o-mini 分析摘要
└── notifications/
    └── telegram.py    # Telegram Bot 推送
```

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| GET | `/api/stocks/` | 查看自选股 |
| POST | `/api/stocks/` | 添加股票 |
| DELETE | `/api/stocks/{ticker}` | 移除股票（软删除） |
| POST | `/api/stocks/scan` | 手动触发扫描 |
| GET | `/api/signals/` | 查看信号（支持 limit/level 过滤） |
| GET | `/api/signals/{ticker}` | 查看指定股票信号 |

## 技术栈

- **后端**：Python 3.12, FastAPI, APScheduler
- **数据**：yfinance（行情）, pandas-ta（技术指标）
- **LLM**：GPT-4o-mini via Sub2API 代理
- **通知**：Telegram Bot API（httpx）
- **数据库**：TiDB Serverless + SQLAlchemy（MySQL 兼容）
- **部署**：Docker Compose
