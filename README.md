# Stock Signal Monitor

> 美股技术信号自动扫描 + Telegram 推送。每日收盘后自动分析，**仅当多指标共振时**推送 GPT-4o-mini 生成的中文分析摘要。解决"没空盯盘"的问题。

## 为什么做这个

单一技术指标的胜率只有 40–60%，信号太多、噪音太大。这个工具的核心逻辑是**信号共振**：

- 单指标触发 → 仅记录，不打扰你
- **2+ 指标同向共振 → 才推 Telegram**，附带 GPT-4o-mini 中文分析

一天最多收到几条真正值得关注的信号。

---

## 功能

| 功能 | 说明 |
|------|------|
| 📡 自动扫描 | 每日 17:00 ET（美股收盘后）触发，无需手动操作 |
| 🔍 多指标检测 | MACD、RSI、均线交叉（20/50 EMA）、布林带 |
| ⚡ 共振过滤 | 2+ 指标同向 → STRONG，单指标 → WEAK，布林带单独 → WATCH |
| 🤖 LLM 摘要 | GPT-4o-mini 生成：综合判断 + 操作价位 + 关键理由（≤150字） |
| 📱 Telegram 推送 | 仅推 STRONG 且置信度 ≥ 60 的信号 |
| 🗄️ 历史记录 | 所有信号（含 WEAK/WATCH）入库，便于回溯分析 |
| 🖥️ Swagger UI | 无前端，通过 `/docs` 管理自选股、查看信号、手动触发 |

## 快速上手

### 方式一：Docker（推荐）

```yaml
# docker-compose.yml
services:
  backend:
    image: ghcr.io/nianyi778/stock-signal-monitor:1.0.0
    ports:
      - "8000:8000"
    environment:
      DATABASE_URL: mysql+pymysql://user:pass@host:4000/db?ssl_mode=VERIFY_IDENTITY
      TELEGRAM_BOT_TOKEN: your_bot_token
      TELEGRAM_CHAT_ID: your_chat_id
      OPENAI_API_KEY: sk-...
      OPENAI_BASE_URL: https://sub2api.nianyi.dpdns.org/v1
      SCHEDULER_CRON_HOUR: "17"
      PUSH_MIN_CONFIDENCE: "60"
    restart: unless-stopped
```

```bash
docker compose up -d
```

```bash
pip install -r requirements.txt
cp .env.example .env  # 填入你的配置
uvicorn app.main:app --reload
```

打开 [http://localhost:8000/docs](http://localhost:8000/docs) 开始使用。

## 使用流程

**1. 添加自选股**

```http
POST /api/stocks/
{"ticker": "AAPL"}
```

**2. 手动触发扫描（测试）**

```http
POST /api/stocks/scan
```

**3. 查看信号**

```http
GET /api/signals/?level=STRONG    # 仅强信号
GET /api/signals/NVDA             # 指定股票
GET /api/signals/?limit=20        # 最近20条
```

之后每天 17:00 ET 自动扫描，有共振信号时 Telegram 收到推送。

## 信号分级逻辑

```
MACD 上穿零轴                     →  WEAK BUY
RSI 超卖 (<30)                    →  WEAK BUY
MACD + RSI 同时触发               →  STRONG BUY  ← 推送
                                      indicator: "MACD+RSI"
                                      confidence: max(单指标) + 20 加成

布林带突破上/下轨                  →  WATCH       ← 仅记录
任意信号 confidence < 60          →  不推送
```

**置信度计算：**
- MACD：柱状图绝对值 × 1000（反映动能强弱）
- RSI：`|RSI - 50| × 2`（偏离中线越远越强）
- 均线交叉：快慢线距离百分比 × 10
- 共振加成：+20，上限 95

## 环境变量

| 变量 | 必填 | 默认 | 说明 |
|------|------|------|------|
| `DATABASE_URL` | ✅ | — | TiDB Serverless 连接串（MySQL 兼容） |
| `TELEGRAM_BOT_TOKEN` | ✅ | — | 从 [@BotFather](https://t.me/botfather) 获取 |
| `TELEGRAM_CHAT_ID` | ✅ | — | 你的 Telegram Chat ID |
| `OPENAI_API_KEY` | ✅ | — | OpenAI / Sub2API Key |
| `OPENAI_BASE_URL` | — | `https://sub2api.nianyi.dpdns.org/v1` | LLM 代理地址 |
| `SCHEDULER_CRON_HOUR` | — | `17` | 每日扫描时间（美东时区） |
| `PUSH_MIN_CONFIDENCE` | — | `60` | 推送置信度阈值（0–100） |

> **东京时区参考**：`SCHEDULER_CRON_HOUR=17`（ET）≈ 早上 6:00–7:00 JST

## API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/health` | 健康检查 |
| `GET` | `/api/stocks/` | 查看自选股列表 |
| `POST` | `/api/stocks/` | 添加股票到自选 |
| `DELETE` | `/api/stocks/{ticker}` | 移除股票（软删除） |
| `POST` | `/api/stocks/scan` | 手动触发扫描 |
| `GET` | `/api/signals/` | 信号列表（`limit` / `level` 过滤） |
| `GET` | `/api/signals/{ticker}` | 指定股票的信号历史 |

## 技术栈

- **后端**：Python 3.12 · FastAPI · APScheduler
- **行情数据**：yfinance
- **技术指标**：pandas-ta（MACD / RSI / EMA / Bollinger Bands）
- **LLM**：GPT-4o-mini（via Sub2API 代理，~$0.015/天）
- **通知**：Telegram Bot API（httpx，无 SDK 依赖）
- **数据库**：TiDB Serverless + SQLAlchemy（MySQL 兼容，零运维）
- **CI/CD**：GitHub Actions → ghcr.io（push tag 自动构建 amd64 + arm64）
