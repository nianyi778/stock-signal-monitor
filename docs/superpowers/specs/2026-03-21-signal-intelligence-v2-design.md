# Signal Intelligence V2 — Design Spec

**Date:** 2026-03-21
**Status:** Approved
**Goal:** 系统给出的建议操盘人直接信任就能拿到跑赢大盘的胜率

---

## 背景与目标

当前系统（v1.4.0）已能检测技术信号并推送 Telegram，但存在以下问题：
- 打分权重是手工设定，未经回测验证
- 只给入场建议，缺少目标价、止损价、持仓期间监控
- 无法量化验证准确率，也无法自我优化

**目标用户：** 波段操作为主（1~4周），接受中长线，目标跑赢 S&P500。
**成功标准：** 系统建议的期望值 > SPY 同期表现，风险回报比 ≥ 1:1.5。

## 子系统一：完整信号输出（进 / 跑 / 割）

### 输出格式

每条强信号推送包含：

```
🟢 NVDA — 多指标共振，建议入场

📥 进场区间:  $875 ~ $890
   （支撑上方回踩，当前价 $882）

🎯 目标价:   $950  (+7.7%)
   （布林上轨 + 20日高点阻力）

🛑 止损价:   $845  (-4.2%)
   （MA50 下方 3%）

📊 风险回报:  1 : 1.8
⏱ 预计持仓:  2~4 周（波段）

━━ 持仓期间监控 ━━
  ⚠️  跌破 $860 → 预警，关注
  🔴  跌破 $845 → 止损线，割
  ✅  触及 $930 → 分批止盈区间
```

### 价格计算规则

| 字段 | 计算方式 |
|------|---------|
| 进场区间下限 | 最近支撑位上方 0.2% |
| 进场区间上限 | 当前价上方 1% |
| 目标价 | 当前价上方最近阻力位（布林上轨 / 20日高点 / 52周高） |
| 止损价 | 最近支撑位下方 3% |
| 预警线 | 止损价上方 1.5%（给操作时间） |
| 分批止盈 | 目标价的 95%（提前布局） |
| 风险回报比 | (目标价 - 入场中间价) / (入场中间价 - 止损价) |
| 预计持仓 | 固定返回"2~4周（波段）" |

### 持仓监控

新增数据库表 `ActiveTrade`：记录每条已推送信号的入场价、目标价、止损价、状态。

每日收盘后调度器额外检查：
1. 查所有 `status=ACTIVE` 的 ActiveTrade
2. 获取当日收盘价
3. 触发止损 → 推 Telegram "🔴 止损提醒" + 标记 `status=STOPPED`
4. 触及目标价 → 推 Telegram "✅ 目标达成" + 标记 `status=TARGET_HIT`
5. 触及预警线 → 推 Telegram "⚠️ 预警" （不关闭仓位）

## 子系统二：回测引擎

### 数据流

```
yfinance 历史日线 (2年)
  → 在每个 T 时间点运行 run_signals(ticker, data[:T])
  → 记录: 信号方向、评分、入场价、目标价、止损价
  → 向后看 20 交易日，判断结果 (WIN / LOSS / NEUTRAL)
  → 同期对比 SPY 涨跌 → 计算超额收益
  → 对每组权重参数记录胜率和期望值
```

### 权重优化

使用网格搜索遍历权重组合：

```python
param_grid = {
    "strong_signal_w": [2, 3, 4],
    "weak_signal_w":   [0.5, 1, 1.5],
    "rsi_w":           [1, 1.5, 2],
    "ma_position_w":   [0.3, 0.5, 0.8],
    "analyst_w":       [0.5, 1, 1.5],
}
```

选出期望值（胜率 × 平均盈利 - 败率 × 平均亏损）最高的组合。

### 输出报告

```
回测结果 (2024-01 ~ 2025-12)
  信号总数: 312  |  胜: 189  |  败: 87  |  持平: 36
  胜率: 60.6%    |  平均盈利: +7.8%  |  平均亏损: -4.0%
  期望值: +3.1%/笔
  vs SPY 同期: +1.6% 超额收益
  最优权重: { strong: 3.5, rsi: 1.8, ma: 0.5, analyst: 0.7 }
```

### 触发方式

- 手动：Telegram Bot "📊 运行回测" 按钮
- 自动：每月 1 日凌晨自动运行，更新权重

## 子系统三：自学习循环

### 架构

```
每日调度器
  ├─ 持仓监控（检查止损/目标）
  └─ 每月自动回测（第 1 天）
       ├─ 跑回测 → 找最优权重
       ├─ 写入 DB 表 signal_weights（带时间戳）
       └─ 权重变化 > 20% → 推 Telegram 通知
            "系统已更新权重，期望值 +3.1% → +3.8%"

每次 run_signals() 执行时
  └─ 从 DB 读取最新权重（热加载，无需重启）
```

### 数据库新增

| 表 | 字段 | 说明 |
|----|------|------|
| `active_trades` | id, ticker, signal_id, entry_low, entry_high, target, stop, warn, status, opened_at, closed_at | 持仓记录 |
| `signal_outcomes` | id, signal_id, ticker, direction, entry_price, exit_price, outcome, holding_days, vs_spy | 结果追踪 |
| `signal_weights` | id, version, weights_json, backtest_winrate, backtest_expectancy, created_at | 权重版本 |

## 技术边界

- **数据源：** yfinance 全量免费，不需要额外 API
- **回测频率：** 月度（不做日内，日线信号已够）
- **过拟合防护：** 回测用最近 2 年，每次滚动更新（不用全历史）
- **基准：** SPY（标普500 ETF），超额收益 = 信号收益 - SPY 同期收益
- **最小样本：** 少于 30 条有效信号不更新权重（防止小样本失真）

## 实现顺序

1. **完整信号输出** — 修改 `engine.py` + `analysis.py` + `telegram.py`，加 `ActiveTrade` 表
2. **持仓监控** — 扩展 `scheduler.py` 日常检查逻辑
3. **回测引擎** — 新增 `app/backtest/engine.py` + `app/backtest/optimizer.py`
4. **自学习循环** — 扩展 scheduler + `signal_weights` 表热加载
5. **Telegram 入口** — 加"运行回测"按钮 + 权重更新推送
